API - Entrega Julio IU

Documentación de clases, métodos, parámetros y retornos del sistema de pruebas.

1. Descripción general

Esta aplicación permite ejecutar pruebas automáticas sobre entidades definidas mediante estructuras JavaScript. Cada entidad dispone de una estructura, definiciones de test de atributos, pruebas concretas y pruebas de submit.

Las entidades incluidas en esta entrega son:

  • project
  • analysis_preparation
  • characteristic

El sistema comprueba pruebas de atributos y pruebas de formulario para las acciones:

  • ADD
  • EDIT
  • SEARCH

2. Estructura general de una entidad

Cada entidad debe tener una variable global con el formato: 

var nombreentidad_estructura = {
  entity: "nombreentidad",
  attributes: [
    {
      name: "nombre_atributo",
      type: "varchar",
      element: "input",
      required: {
        ADD: true,
        EDIT: true,
        SEARCH: false
      },
      validations: {
        ADD: [
          { method: "min_size", param: 1, error: "campo_min_size_KO" }
        ],
        EDIT: [
          { method: "min_size", param: 1, error: "campo_min_size_KO" }
        ],
        SEARCH: [
          { method: "format", param: "^[A-Za-z]*$", error: "campo_format_KO" }
        ]
      }
    }
  ]
};
Propiedad Descripción
entity Nombre de la entidad.
attributes Listado de atributos que forman la entidad.
name Nombre del atributo.
type Tipo lógico del atributo: int, varchar, date, enum o file.
element Elemento HTML asociado: input, textarea, select, file o inputfile.
required Indica si el campo es obligatorio en ADD, EDIT o SEARCH.
validations Validaciones que se ejecutan por cada acción.

3. Flujo general de ejecución

index.html
→
menu_entidades.js
→
Gestor
→
TestForm / TestSubmit
→
Validations
→
Ventana de resultados

El usuario selecciona una entidad en el menú. La clase Gestor comprueba que existan los ficheros y variables necesarias. Si la entidad es válida, muestra los botones para ejecutar el test de atributos o el test de formulario.

4. Clase Gestor

Clase encargada de gestionar la selección de entidades y comprobar que existen las estructuras necesarias.

Método Parámetros Retorno Descripción
constructor() No recibe parámetros. Objeto Gestor. Inicializa el gestor principal de la aplicación.
cargarEntidad(nombreEntidad) nombreEntidad: string No devuelve valor. Comprueba la entidad seleccionada y muestra los botones de test.
comprobarEntidad(nombreEntidad) nombreEntidad: string true o array de errores. Comprueba que existan las variables globales de estructura, test de atributos y test de submit.
mostrarError(mensaje) mensaje: string No devuelve valor. Muestra un mensaje de error en modal o en pantalla.
crearBotones(nombreEntidad) nombreEntidad: string No devuelve valor. Crea los botones para ejecutar Test de atributos y Test de formulario.

5. Clase TestForm

Clase encargada de ejecutar las pruebas de atributos de una entidad.

TestForm
→
analizar estructura
→
analizar definiciones
→
ejecutar pruebas
→
ventana resumen
→
ventana detalle
Método Parámetros Retorno Descripción
constructor(nombreEntidad) nombreEntidad: string Objeto TestForm. Recibe el nombre de la entidad y carga estructura, definiciones y pruebas.
ejecutar() No recibe parámetros. No devuelve valor. Ejecuta el análisis de estructura, definiciones y pruebas de atributos.
crearInstanciaEntidad() No recibe parámetros. Objeto de entidad o null. Crea una instancia de la clase de entidad para poder ejecutar validaciones personalizadas.
analizarEstructura() No recibe parámetros. Objeto con información de estructura. Comprueba si la estructura de la entidad es correcta.
analizarDefinicionesTest() No recibe parámetros. Objeto resumen. Cuenta definiciones de test, definiciones correctas, incorrectas y número de definiciones por atributo.
definicionTestCorrecta(definicion) definicion: array boolean Comprueba si una definición de test está bien definida a nivel de tipos.
analizarPruebas(resultadosPruebas) resultadosPruebas: array Objeto resumen. Cuenta pruebas, pruebas correctas, incorrectas, pruebas de error y pruebas de éxito.
pruebaCorrectaTipo(prueba) prueba: array boolean Comprueba si una prueba está bien definida a nivel de tipos.
ejecutarPruebas() No recibe parámetros. Array de resultados. Ejecuta todas las pruebas de atributos de la entidad.
ejecutarUnaPrueba(prueba) prueba: array Objeto resultado. Ejecuta una prueba individual y compara resultado esperado con obtenido.
extraerValoresPrueba(campo, valoresOriginales) campo: string, valoresOriginales: object/array Objeto de valores. Transforma los valores de prueba al formato utilizado por el motor.
obtenerAtributo(nombreCampo) nombreCampo: string Objeto atributo o null. Busca un atributo dentro de la estructura de la entidad.
abrirVentanaResumen(...) Objetos de resumen y resultados. No devuelve valor. Abre la ventana principal de resultados del test de atributos.
escapeHTML(valor) valor: cualquier valor string Escapa caracteres HTML para mostrar datos en ventanas externas.

Clase encargada de ejecutar pruebas de formulario completas para ADD, EDIT y SEARCH.

→

→

Método Parámetros Retorno Descripción
constructor(nombreEntidad) nombreEntidad: string Objeto TestSubmit. Carga estructura, pruebas físicas, pruebas de atributos y validador.
ejecutar() No recibe parámetros. No devuelve valor. Ejecuta pruebas físicas y generadas automáticamente.
crearInstanciaEntidad() No recibe parámetros. Objeto de entidad o null. Crea la instancia de entidad para validaciones personalizadas.
CheckSubmit(accion, valores) accion: string, valores: object true o array de errores. Valida todos los campos de un formulario para ADD, EDIT o SEARCH.
valorEstaVacio(valor) valor: cualquier valor boolean Indica si un valor debe considerarse vacío.
ejecutarUnaPruebaSubmit(prueba, origen) prueba: array, origen: string Objeto resultado. Ejecuta una prueba de submit física o generada.
compararResultadoSubmit(esperado, obtenido) esperado: any, obtenido: any boolean Compara errores esperados con errores obtenidos.
generarResumenPorAccion(resultados) resultados: array Objeto resumen. Agrupa las pruebas de submit por ADD, EDIT y SEARCH.
generarSubmitAutomatico() No recibe parámetros. Array de pruebas generadas. Genera pruebas de submit a partir de las pruebas de atributos.
obtenerValoresCorrectosBase(accion) accion: string Objeto de valores correctos. Devuelve un formulario base correcto para la acción indicada.

extraerValoresPrueba(campo, valoresOriginales) campo: string, valoresOriginales: object/array Objeto de valores. Normaliza los valores de las pruebas de atributos para usarlos en submit.
abrirVentanaResumenSubmit(resumen, resultados, pruebasGeneradas) resumen: object, resultados: array, pruebasGeneradas: array No devuelve valor. Abre la ventana de resumen de submit, detalle y JSON generado.

7. Clase ValidateFieldsForm

Clase puente entre la estructura de una entidad y la clase Validations. Crea campos temporales ocultos para ejecutar las validaciones sin depender de formularios reales.

Método Parámetros Retorno Descripción
constructor() No recibe parámetros. Objeto ValidateFieldsForm. Inicializa una instancia de Validations.
limpiarCamposTemporales() No recibe parámetros. No devuelve valor. Elimina los campos temporales creados para pruebas.
prepararCampos(estructura, valores) estructura: object, valores: object No devuelve valor. Crea todos los campos temporales de una entidad.
prepararCampo(campo, elemento, valor) campo: string, elemento: string, valor: any Elemento HTML. Crea un input, textarea, select o fichero temporal.
ejecutarValidacionesAtributo(atributo, accion, instanciaEntidad) atributo: object, accion: string, instanciaEntidad: object true o código de error. Ejecuta las validaciones de un atributo para una acción.
ejecutarValidacion(atributo, validacion, instanciaEntidad) atributo: object, validacion: object, instanciaEntidad: object true o código de error. Ejecuta una validación concreta.
ejecutarValidacionPersonalizada(atributo, instanciaEntidad, error) atributo: object, instanciaEntidad: object, error: string true o código de error. Ejecuta métodos personalizados del tipo nombreCampo_personalized_validation().

8. Clase Validations

Clase que contiene las validaciones atómicas utilizadas por TestForm y TestSubmit.

Método Parámetros Retorno Descripción
constructor() No recibe parámetros. Objeto Validations. Inicializa la clase de validaciones.
getElement(id) id: string Elemento HTML o null. Obtiene un elemento del DOM por su id.
getValue(id) id: string string Obtiene el valor de un input, textarea, select o campo temporal.
getFile(id) id: string File, objeto simulado o null. Obtiene el fichero real o simulado asociado a un input file.
min_size(id, minsize) id: string, minsize: number boolean Comprueba que el valor tenga longitud mínima.
max_size(id, maxsize) id: string, maxsize: number boolean Comprueba que el valor no supere la longitud máxima.
format(id, exprreg) id: string, exprreg: string boolean Comprueba el valor mediante una expresión regular.
value(id, valoresPermitidos) id: string, valoresPermitidos: array boolean Comprueba que el valor esté dentro de un conjunto permitido.
value_or_empty(id, valoresPermitidos) id: string, valoresPermitidos: array boolean Permite valor vacío o un valor incluido en el conjunto permitido.
date_format(id) id: string boolean Comprueba que la fecha tenga formato dd/mm/aaaa y sea real.
date_format_or_empty(id) id: string boolean Permite fecha vacía o fecha válida en formato dd/mm/aaaa.
not_exist_file(id) id: string boolean Comprueba si existe un fichero asociado al campo.
fichero(id) id: string boolean Alias de not_exist_file(id). Comprueba que exista un fichero.
max_size_file(id, maxsize) id: string, maxsize: number boolean Comprueba que el tamaño del fichero no supere el máximo indicado.
type_file(id, array_tipos) id: string, array_tipos: array boolean Comprueba que el tipo MIME del fichero esté permitido.
format_name_file(id, exprreg) id: string, exprreg: string boolean Comprueba el nombre del fichero mediante expresión regular.
max_size_file_optional(id, maxsize) id: string, maxsize: number boolean Si no hay fichero devuelve true; si hay fichero comprueba el tamaño.
type_file_optional(id, array_tipos) id: string, array_tipos: array boolean Si no hay fichero devuelve true; si hay fichero comprueba el tipo.
format_name_file_optional(id, exprreg) id: string, exprreg: string boolean Si no hay fichero devuelve true; si hay fichero comprueba el nombre.
date_greater_than(idFechaFin, idFechaInicio) idFechaFin: string, idFechaInicio: string boolean Comprueba que una fecha final sea posterior a una fecha inicial.

9. Validaciones personalizadas

Las validaciones personalizadas permiten comprobar reglas que dependen de varios campos.

En esta entrega se usa una validación personalizada en la entidad project: 

end_date_project_personalized_validation()

Esta validación comprueba que end_date_project sea posterior a start_date_project.

Método Parámetros Retorno Descripción
end_date_project_personalized_validation() No recibe parámetros directos. true o end_date_project_date_greater_than_KO Comprueba que la fecha final del proyecto sea posterior a la fecha inicial.

10. Formato de los tests de atributos

Las definiciones de test usan la variable: 

var nombreentidad_def_tests = Array(...)

Las pruebas concretas usan la variable: 

var nombreentidad_pruebas = Array(...)

Formato de una definición de test

[
  "entidad",
  "campo",
  "elemento",
  numeroDefinicion,
  "descripcion",
  "ADD/EDIT/SEARCH",
  resultadoEsperado,
  "mensaje"
]

Formato de una prueba

[
  "entidad",
  "campo",
  numeroDefinicion,
  numeroPrueba,
  "ADD/EDIT/SEARCH",
  { campo: "valor" },
  resultadoEsperado
]

11. Formato de los tests de submit

Las pruebas de submit usan la variable: 

var nombreentidad_TestSubmit = Array(...)

Formato

[
  "entidad",
  "ADD/EDIT/SEARCH",
  numeroPrueba,
  "descripcion",
  {
    campo1: "valor",
    campo2: "valor"
  },
  resultadoEsperado
]

El sistema también genera automáticamente pruebas de submit a partir de las pruebas de atributos. Estas pruebas se muestran en una ventana externa en formato JSON copiable.

12. Resultados esperados del sistema

Entidad Test de atributos Test de formulario
project ADD, EDIT y SEARCH con 0 incorrectas. Resumen por acción con 0 incorrectas.
analysis_preparation ADD, EDIT y SEARCH con 0 incorrectas. Resumen por acción con 0 incorrectas.
characteristic ADD, EDIT y SEARCH con 0 incorrectas. Resumen por acción con 0 incorrectas.
